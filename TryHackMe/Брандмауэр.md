Брандмауэр — это устройство в сети, отвечающее за определение того, какой трафик может входить в сеть и выходить из неё. Представьте, что брандмауэр — это пограничная служба безопасности для сети. Администратор может настроить брандмауэр на **разрешение** или **запрет** входящего или исходящего трафика в зависимости от множества факторов, таких как:

- Откуда поступает трафик? (был ли задан брандмауэру параметр для принятия/отклонения трафика из определенной сети?)
- Куда направляется трафик? (получил ли брандмауэр указание принимать/запрещать трафик, предназначенный для определенной сети?)
- Для какого порта предназначен трафик? (был ли задан брандмауэру параметр для принятия/отклонения трафика, предназначенного только для порта 80?)
- Какой протокол используется для передачи данных? (указано ли в брандмауэре, что он должен принимать/запрещать трафик по UDP, TCP или по обоим протоколам?)

Брандмауэры выполняют проверку пакетов данных, чтобы получить ответы на эти вопросы.

Брандмауэры бывают разных форм и размеров. От специализированных аппаратных средств (которые часто используются в крупных сетях, например на предприятиях) для обработки больших объёмов данных до бытовых маршрутизаторов (например, у вас дома!) или программного обеспечения, такого как [Snort](https://www.snort.org/). Брандмауэры можно разделить на 2–5 категорий.

В таблице ниже мы рассмотрим две основные категории брандмауэров.

| **КатегорияБрандмауэра** | **Описание**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Stateful                 | Этот тип брандмауэра использует всю информацию о соединении. Вместо того чтобы проверять каждый пакет, этот брандмауэр определяет поведение устройства **на основе всего соединения**.<br><br>Этот тип брандмауэра потребляет больше ресурсов по сравнению с брандмауэрами без сохранения состояния, поскольку решения принимаются динамически. Например, брандмауэр может разрешить первые этапы TCP-рукопожатия, которые впоследствии завершатся ошибкой.<br><br>Если соединение с хостом будет плохим, это заблокирует работу всего устройства.                                                                                                                                                              |
| Stateless                | Этот тип брандмауэра использует статический набор правил для определения того, являются ли **отдельные пакеты** допустимыми или нет. Например, если устройство отправило некорректный пакет, это не обязательно означает, что всё устройство будет заблокировано.<br><br>Хотя эти брандмауэры потребляют гораздо меньше ресурсов, чем их аналоги, они гораздо «глупее». Например, эти брандмауэры эффективны только в том случае, если в них заданы правила. Если правило не соответствует точному запросу, оно фактически бесполезно.<br><br>Однако эти брандмауэры отлично справляются с приёмом больших объёмов трафика от множества хостов (например, при распределённой атаке типа «отказ в обслуживании») |
