Давайте рассмотрим некоторые из этих протоколов подробнее:

| **Протокол**                                                          | **Номер порта** | **Описание**                                                                                                                                                                    |
| --------------------------------------------------------------------- | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **F**ile **T**ransfer **P**rotocol (**FTP**)                          | 21              | Этот протокол используется приложением для обмена файлами, построенным по модели «клиент-сервер», то есть вы можете загружать файлы из центрального хранилища.                  |
| **S**ecure **Sh**ell (**SSH**)                                        | 22              | Этот протокол используется для безопасного входа в системы через текстовый интерфейс управления.                                                                                |
| **H**yper**T**extTransferProtocol протокол передачи данных (**HTTP**) | 80              | Этот протокол обеспечивает работу Всемирной паутины (WWW)! Ваш браузер использует его для загрузки текста, изображений и видео с веб-страниц.                                   |
| **H**yper**T**ext **T**ransfer **P**rotocol **S**ecure (**HTTPS**)    | 443             | Этот протокол делает то же самое, что и предыдущий, но с использованием безопасного шифрования.                                                                                 |
| **S**erver **M**essage **B**lock (**SMB**)                            | 445             | Этот протокол похож на протокол передачи файлов (FTP); однако, помимо файлов, SMB позволяет совместно использовать такие устройства, как принтеры.                              |
| **R**emote **D**esktop **P**rotocol (**RDP**)                         | 3389            | Этот протокол представляет собой безопасное средство входа в систему с использованием визуального интерфейса рабочего стола (в отличие от текстовых ограничений протокола SSH). |

Мы лишь вкратце рассмотрели наиболее распространённые протоколы в сфере кибербезопасности. Дополнительную информацию можно [найти в таблице с перечислением 1024 распространённых портов](http://www.vmaxx.net/techinfo/ports.htm).

Здесь стоит отметить, что эти протоколы соответствуют только стандартам. То есть вы можете администрировать приложения, взаимодействующие с этими протоколами, через порт, отличный от стандартного (например, запустить веб-сервер на порту 8080 вместо стандартного порта 80). Однако обратите внимание, что приложения будут считать, что соблюдается стандарт, поэтому вам нужно будет указать **двоеточие (:)** вместе с номером порта.
