**Что такое HTTP? (протокол передачи гипертекста)**

HTTP — это протокол, который используется при просмотре веб-сайтов. Он был разработан Тимом Бернерсом-Ли и его командой в период с 1989 по 1991 год. HTTP — это набор правил, используемых для взаимодействия с веб-серверами при передаче данных веб-страниц, будь то HTML, изображения, видео и т. д.

**Что такое HTTPS?** ****(Защищённый протокол передачи гипертекста)****  

HTTPS — это безопасная версия HTTP. Данные HTTPS зашифрованы, поэтому они не только недоступны для просмотра, но и гарантируют, что вы общаетесь с нужным веб-сервером, а не с его имитацией.

# **Запросы и ответы**
Uniform Resource Locator) - URL
URL-адрес — это, по сути, инструкция о том, как получить доступ к ресурсу в Интернете. На изображении ниже показано, как выглядит URL-адрес со всеми его функциями (не все функции используются в каждом запросе).

![На схеме показаны различные части URL-адреса на примере, где http — это схема, user:password — это пользователь, tryhackme.com — это домен или хост, 80 — это порт, view-room — это путь, ?id=1 — это строка запроса, а #task3 — это фрагмент.  Полный адрес выглядит так: http://user:password@tryhackme.com:80/view-room?id=1#task3.](https://tryhackme-images.s3.amazonaws.com/user-uploads/5c549500924ec576f953d9fc/room-content/34ad66d8b90aaaa35f9536d3b152ea97.png)

**Схема:** Указывает, какой протокол использовать для доступа к ресурсу, например HTTP, HTTPS, FTP (протокол передачи файлов).

**Пользователь:** Для входа в некоторые сервисы требуется аутентификация. Чтобы войти, можно указать имя пользователя и пароль в URL-адресе.

**Хост:** Доменное имя или IP-адрес сервера, к которому вы хотите получить доступ.

**Порт:** Порт, к которому вы собираетесь подключиться. Обычно это 80 для HTTP и 443 для HTTPS, но он может быть привязан к любому порту в диапазоне от 1 до 65535.

**Путь:** Имя файла или расположение ресурса, к которому вы пытаетесь получить доступ.

**Строка запроса:** Дополнительная информация, которую можно отправить по запрошенному пути. Например, /blog?**id=1** укажет пути к блогу, что вы хотите получить статью с идентификатором 1.

**Фрагмент:** Это ссылка на место на запрашиваемой странице. Обычно это используется на страницах с большим объемом контента, и определенная часть страницы может быть напрямую связана с этим местом, чтобы пользователь мог увидеть его сразу после перехода на страницу.


HTTP-методы — это способ, с помощью которого клиент указывает предполагаемое действие при отправке HTTP-запроса. Существует множество HTTP-методов, но мы рассмотрим самые распространённые, хотя в основном вам придётся иметь дело с методами GET и POST.

**GET Request**

Это используется для получения информации с веб-сервера.  

**POST Request**

Это используется для отправки данных на веб-сервер и, возможно, для создания новых записей  

**PUT Request**

Это используется для отправки данных на веб-сервер для обновления информации

**Запрос на УДАЛЕНИЕ (DELETE)**  

Это используется для удаления информации/записей с веб-сервера.


 HTTP-сервера первая строка всегда содержит код состояния, информирующий клиента о результатах запроса, а также о том, как с ними работать. Эти коды состояния можно разделить на 5 различных диапазонов:

|                                                  |                                                                                                                                                                                        |
| ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **100–199 — информационный ответ (Information)** | Они отправляются для того, чтобы сообщить клиенту, что первая часть его запроса принята и ему следует продолжить отправку остальной части запроса. Эти коды уже не так распространены. |
| **200–299 — успех (Success)**                    | Этот диапазон кодов состояния используется для того, чтобы сообщить клиенту, что его запрос был успешно выполнен.                                                                      |
| **300–399 — перенаправление (Redirection)**      | Они используются для перенаправления запроса клиента на другой ресурс. Это может быть другая веб-страница или вообще другой сайт.                                                      |
| **400–499 — ошибки клиента (Client errors)**     | Используется для информирования клиента о том, что в его запросе была допущена ошибка.                                                                                                 |
| **500–599 — ошибки сервера (Server errors)**     | Этот код предназначен для ошибок, возникающих на стороне сервера, и обычно указывает на серьёзную проблему с обработкой запроса сервером.                                              |

**Распространённые HTTP коды состояния:**

Существует множество различных HTTP кодов состояния, не говоря уже о том, что приложения могут определять свои собственные коды. Мы рассмотрим наиболее распространённые HTTP ответы, с которыми вы, скорее всего, столкнётесь:

|                                                             |                                                                                                                                                                                                              |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **200 — ОК**                                                | Запрос был успешно выполнен.                                                                                                                                                                                 |
| **201 — Создано (Created)**                                 | Создан ресурс (например, новый пользователь или новая запись в блоге).                                                                                                                                       |
| **301 — Перенаправление (Moved Permanently)**               | Это перенаправляет браузер клиента на новую веб-страницу или сообщает поисковым системам, что страница переместилась в другое место и их следует искать там.                                                 |
| **302 — найдено (Found)**                                   | Это похоже на описанный выше постоянный редирект, но, как следует из названия, это временное изменение, которое может быть отменено в ближайшем будущем.                                                     |
| **400 — неверный запрос (Bad request)**                     | Это сообщает браузеру, что в запросе что-то не так или чего-то не хватает. Иногда это может использоваться, если запрашиваемый ресурс веб-сервера ожидает определённый параметр, который клиент не отправил. |
| **401 — несанкционированный доступ (Not Authorised)**       | В настоящее время вам не разрешено просматривать этот ресурс, пока вы не авторизуетесь в веб-приложении, чаще всего с помощью имени пользователя и пароля.                                                   |
| **403 — запрещено (Forbidden)**                             | У вас нет разрешения на просмотр этого ресурса, независимо от того, вошли вы в систему или нет.                                                                                                              |
| **405 — метод не разрешён (Method Not Allowed)**            | Ресурс не поддерживает этот метод запроса. Например, вы отправляете GET-запрос к ресурсу /create-account, в то время как он ожидает POST-запрос.                                                             |
| **404 — страница не найдена (Page Not Found)**              | Запрошенная вами страница/ресурс не существует.                                                                                                                                                              |
| **500 — внутренняя ошибка службы (Internal Service Error)** | Сервер обнаружил в вашем запросе ошибку, которую он не может обработать должным образом.                                                                                                                     |
| **503 — служба недоступна (Service Unavailable)**           | Этот сервер не может обработать ваш запрос, так как он либо перегружен, либо отключен на время технического обслуживания.                                                                                    |

Если вы лучше воспринимаете информацию визуально, обратите внимание на отличный ресурс [http.cat](https://http.cat/) для изучения кодов состояния. Теперь нажмите кнопку «Просмотреть сайт» справа, чтобы увидеть, как некоторые из этих HTTP сообщений о состоянии выглядят в браузере.
# **HEADERS**
Заголовки — это дополнительные данные, которые можно отправлять на веб-сервер при выполнении запросов.

Хотя при отправке HTTP-запроса заголовки не являются обязательными, без них вам будет сложно нормально просматривать веб-сайт.

**Общие Заголовки запросов**

﻿Это заголовки, которые отправляются с клиента (обычно это ваш браузер) на сервер.  

**Хост:** на некоторых веб-серверах размещено несколько веб-сайтов, поэтому, указав заголовки хоста, вы можете указать, какой из них вам нужен. В противном случае вы получите доступ к веб-сайту сервера по умолчанию.  

**User-Agent:** Это название вашего браузера и номер его версии. Сообщая веб-серверу название вашего браузера, вы помогаете ему правильно форматировать веб-сайт для вашего браузера, а также указываете, что некоторые элементы HTML, JavaScript и CSS доступны только в определённых браузерах.  

**Content-Length:** при отправке данных на веб-сервер, например в форме, длина содержимого указывает веб-серверу, какой объём данных ожидать в веб-запросе. Таким образом, сервер может быть уверен, что не пропустит ни одного фрагмента данных.

**Accept-Encoding:** сообщает веб-серверу, какие методы сжатия поддерживает браузер, чтобы можно было уменьшить размер данных для передачи через Интернет.

  

**Файл cookie:** данные, отправленные на сервер для сохранения вашей информации (дополнительную информацию см. в задании «Файлы cookie»).  

**Общие Заголовки ответов**

Это заголовки, которые сервер возвращает клиенту после запроса.

**Set-Cookie:** информация для сохранения, которая отправляется обратно на веб-сервер при каждом запросе (дополнительную информацию см. в задаче «Файлы cookie»).  

**Cache-Control:** Как долго следует хранить содержимое ответа в кэше браузера, прежде чем он запросит его снова.  

**Content-Type:** Этот заголовок сообщает клиенту, какой тип данных возвращается: HTML, CSS, JavaScript, изображения, PDF, видео и т. д. Благодаря заголовку Content-Type браузер знает, как обрабатывать данные.  

**Content-Encoding:** Какой метод использовался для сжатия данных, чтобы уменьшить их размер при отправке через Интернет.
